# OCR 发票识别流程说明

## 1. 整体流程

```
上传发票图片
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│ 1. 图像预处理                                               │
│    - 红色印章检测与去除 (HSV颜色空间 + inpainting)           │
│    - PaddleOCR 文档方向矫正                                  │
│    - PaddleOCR 文档展平                                      │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│ 2. PaddleOCR v5 文本识别                                    │
│    - 文本检测 (DBNet) → 定位文字区域                         │
│    - 文本识别 (CRNN) → 识别文字内容                           │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│ 3. 字段提取与校验                                            │
│    - 正则表达式匹配关键信息                                   │
│    - 业务规则校验与修正                                       │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│ 4. 数据校验与清洗                                            │
│    - 金额关系校验: 价税合计 = 金额 + 税额                     │
│    - 日期范围: 2000-2030                                     │
│    - 金额范围: 0 ~ 100万                                     │
│    - 税号格式: 15/18/20位                                   │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│ 5. 输出结构化数据                                            │
└─────────────────────────────────────────────────────────────┘
```

## 2. 字段识别规则

### 2.1 发票号码

**正则表达式**:
```python
r'[发票号No№nO]{1,3}[：:\s]*(\d{8})'  # 限定8位数字
```

**匹配示例**:
- `发票号：66215500` → `66215500`
- `No66215500` → `66215500`

**校验逻辑**:
- 固定 8 位数字
- 排除包含"金额"、"税额"等关键词的行

### 2.2 开票日期

**正则表达式**:
```python
r'(\d{4})[年/-](\d{1,2})[月/-](\d{1,2})[日]?'
r'(\d{4})(\d{2})(\d{2})'
```

**匹配示例**:
- `2016年06月12日` → `2016-06-12`

**校验逻辑**:
- 年份范围: 2000-2030
- 月份: 1-12
- 日期: 1-31
- **统一输出格式**: `YYYY-MM-DD`

### 2.3 金额信息

**正则表达式**:
```python
# 金额 (不含税)
r'(?:金额|不含税金额|货物或应税劳务金额)[：:]\s*[￥¥]?\s*(\d+(?:,\d{3})*(?:\.\d{2})?)'

# 税额
r'(?:税额|税费)[：:]\s*[￥¥]?\s*(\d+(?:,\d{3})*(?:\.\d{2})?)'

# 价税合计
r'(?:价税合计|合计|总金额)[（(]?[大、小]*[￥¥]?\s*(\d+(?:,\d{3})*(?:\.\d{2})?)[）)]?'
```

**匹配示例**:
- `金额：￥2947.18` → `2947.18`
- `税额：¥507.82` → `507.82`
- `价税合计：￥3495.00` → `3495.00`

**备选方案** (当未匹配到明确标签时):
1. 排除干扰项：`备注`、`作废`、`购方`、`销方`、`密文`、`校验码`
2. 提取所有符合金额格式的数字
3. 按大小排序：**价税合计**应为最大，**金额**次之

**校验与修正逻辑**:
```
1. 校验: 价税合计 ≈ 金额 + 税额
2. 如果关系不成立，推算缺失值:
   - 知道总额和金额 → 税额 = 总额 - 金额
   - 知道总额和税额 → 金额 = 总额 - 税额
   - 知道金额和税额 → 总额 = 金额 + 税额
3. 如果金额 > 价税合计 → 交换两者值
```

### 2.4 税号 (纳税人识别号)

**正则表达式**:
```python
# 统一社会信用代码: 18位 (字母+数字)
r'([0-9A-HJ-NPQRTUWXY]{2}\d{6}[0-9A-HJ-NPQRTUWXY]{10})'

# 税务登记号: 15位/20位 (纯数字)
r'\b(\d{15})\b'
r'\b(\d{20})\b'
```

**匹配示例**:
- `91110000123456789X` → `91110000123456789X` (统一社会代码)
- `110XXXXXXXXXXXXX` → `110XXXXXXXXXXXXX` (老税号)

**校验逻辑**:
- 长度: 15位 (老税号) / 18位 (统一社会信用代码) / 20位
- 统一代码前两位为地区代码

### 2.5 销售方/购买方信息

**增强的提取逻辑**:

1. **区域检测**:
   - 检测 `购买方` 关键词 → 进入购买方区域
   - 检测 `销售方` 关键词 → 进入销售方区域

2. **标签匹配**:
   | 标签 | 字段名 |
   |------|--------|
   | 名称 | seller_name / buyer_name |
   | 纳税人识别号 | seller_tax_id / buyer_tax_id |
   | 地址、电话 | seller_address / buyer_address |
   | 开户行及账号 | seller_bank / buyer_bank |

3. **同位置/分离位置处理**:
   - 标签和值在同一行
   - 标签和值分离（值在下一行）
   - 状态机追踪当前字段类型

4. **干扰排除**:
   - 排除 `备注`、`作废`、`校验码`、`密码区`、`发票专用章`、`监制` 等

## 3. 代码实现

### 3.1 核心类

**文件**: `ml/invoice_ocr.py`

```python
class InvoiceOCR:
    def __init__(self, enable_preprocessing: bool = True):
        from paddleocr import PaddleOCR
        self.ocr = PaddleOCR(
            use_doc_orientation_classify=enable_preprocessing,
            use_doc_unwarping=enable_preprocessing,
            use_textline_orientation=enable_preprocessing,
            lang=settings.OCR_LANG
        )
```

### 3.2 关键方法

| 方法 | 功能 |
|------|------|
| `preprocess_image()` | 图像预处理：印章去除 |
| `process_image()` | 处理单张发票图片 |
| `extract_fields()` | 从OCR结果提取字段 |
| `validate_and_fix_amounts()` | 校验并修正金额关系 |
| `extract_invoice_no()` | 提取发票号码(8位) |
| `extract_date()` | 提取日期(统一格式) |
| `extract_amount()` | 提取金额(范围校验) |
| `extract_tax_id()` | 提取税号(支持统一代码) |

## 6. 已知局限性

1. **图像质量**: 低分辨率或模糊图片识别准确率下降
2. **版式多样**: 不同地区/类型的发票版式差异可能影响识别
3. **手写内容**: 手写文字识别效果较差
4. **严重重叠**: 多页完全重叠导致无法分离
