# OCR + LLM 合并优化对比分析

## 1. 优化背景

原流程需要两次模型推理：
1. **OCR 识别**：提取发票结构化数据
2. **LLM 分析**：分析发票真伪

优化后合并为一次推理，同时完成两项任务。

---

## 2. 优化方案

### 原流程（分离模式）
```
图片 → OCR推理(20秒) → 结构化数据 → LLM推理(5秒) → 分析结果
总耗时: ~26秒
```

### 优化流程（合并模式）
```
图片 → 合并推理(14秒) → {结构化数据 + 分析结果}
总耗时: ~14秒
```

---

## 3. 性能对比

| 指标 | 数值 |

| LLM 分析耗时 | 5.12秒 |
| **总耗时** | **26.12秒** |

### 3.2 合并模式（优化后）

### 3.3 对比总结

| 指标 | 分离模式 | 合并模式 | 提升 |
|------|---------|---------|------|
| OCR 耗时 | 20.77秒 | 14.22秒 | ↓31% |
| LLM 耗时 | 5.12秒 | 0秒（已合并） | ↓100% |
| **总耗时** | **26.12秒** | **14.22秒** | **↓46%** |
| 推理次数 | 2次 | 1次 | ↓50% |

---

## 4. 优化原理

### 4.1 时间节省来源

1. **消除重复图像处理**
   - 分离模式：图像编码 2 次
   - 合并模式：图像编码 1 次

2. **消除重复模型前向传播**
   - 分离模式：ViT 编码 2 次
   - 合并模式：ViT 编码 1 次

3. **精简 Prompt**
   - 合并后的 Prompt 更紧凑，减少 token 数

### 4.2 合并后的 Prompt 结构

```json
{
    "ocr": {
        "invoice_no": "发票号码",
        "date": "开票日期",
        "amount": "金额",
        ...
    },
    "analysis": {
        "is_suspicious": false,
        "authenticity_score": 0.95,
        ...
    }
}
```

---

## 5. 配置说明

### 启用合并模式（默认）
```bash
USE_COMBINED_OCR_LLM=true python main.py
```

### 使用分离模式
```bash
USE_COMBINED_OCR_LLM=false python main.py
```

---

## 6. 注意事项

### 6.1 合并模式优势
- 速度快 46%
- 显存占用相同
- 代码更简洁

### 6.2 分离模式优势
- OCR 和分析可独立调整
- 可针对不同任务使用不同参数
- 更灵活的错误处理

### 6.3 建议
- **生产环境推荐合并模式**：性能最优
- **调试阶段可用分离模式**：便于定位问题
